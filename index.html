💡 核心设计哲学：流式优先（Streaming-First）

整个系统基于 TypeScript async function* 异步生成器构建。从 API 响应到工具执行到 UI 渲染，数据以流的形式逐步传递，实现最小延迟。工具在消息流入时即刻异步执行，不等待完整响应。

🔄

Query 循环状态机

核心 query() 函数是一个无限循环的异步生成器，管理消息准备 → API 调用 → 流式工具执行 → 状态转移的完整生命周期。支持 7 种终止原因：completed / prompt_too_long / max_turns / aborted 等。

🛡

三层权限模型

Layer 1：全局策略（5 种权限模式）。Layer 2：个别工具授权（8 源规则继承链）。Layer 3：SDK 审计追踪（permissionDenials 完整记录）。企业级可审计性。

⚠ 安全审计发现：4 个关注点

1. Auto 模式分类器可靠性 — LLM 驱动的权限决策具有非确定性，边界情况可能误判。2. Hook 隔离 — Hooks 运行在主进程中而非独立沙箱。3. Bash 解析边界 — splitCommand_DEPRECATED 标签表明已知局限。4. 配置同步 — 多源沙箱配置可能产生冲突。均为加固机会，非严重缺陷。

Design Patterns

企业级工程实践的 12 个关键设计决策

异步生成器驱动

核心 query() 用 async function* 实现。流式处理（逐个 yield）/ 背压支持 / 错误传播 / 资源清理 / 中止控制 — 一个模式解决五个问题。

Bun bundler 的 feature() 函数实现编译时死代码消除。KAIROS / COORDINATOR_MODE / REACTIVE_COMPACT 等特性在外部构建中完全移除。零运行时开销。

💾

分层缓存架构

L1 LRU 内存缓存（固定大小）→ L2 TTL 缓存（后台刷新，stale-while-revalidate）→ L3 文件监听（热更新强制刷新）。读多写少场景 100x 加速。

🛠

buildTool 工厂模式

所有 60+ 工具通过 buildTool() 统一构建。Fail-closed 默认值 + Zod 输入校验 + 权限检查接口 + 并发声明 + UI 渲染钩子 = 一致的工具行为保证。

📦

环形缓冲 UUID 去重

BoundedUUIDSet 用固定大小环形缓冲 + Set 实现 O(1) 添加/查询/内存。FIFO 驱逐最旧条目。解决 IDE Bridge 的消息回显和历史重放问题。

📊

选择器式状态订阅

useAppState(s => s.mainLoopModel) 只在模型变化时重新渲染，而非整棵状态树变化。配合 DeepImmutable<T> 和 Object.is 比较，实现精确的渲染控制。

🔀

命令式 + 声明式混合

ScrollBox 的高频滚动用命令式 scrollBy() 绕过 React。只在粘性状态变化时走 React 状态更新。双轨制：频繁操作零 React 开销，异常情况才触发重新渲染。

🔐

权限决策可追踪

每个权限决策都携带完整的 PermissionDecisionReason — 规则匹配 / 分类器决策 / Hook 拒绝 / 沙箱覆盖 / 安全检查。用户困惑时系统能清晰解释"为什么被拒绝"。

🌍

Key Insights

核心洞察

512,664 行代码中最值得学习的工程智慧

Architecture 流式优先不是优化，是架构基因

Claude Code 的整个数据管线都基于 async function* 构建。这不是事后优化 — 是从第一天就决定的架构基因。从 API 流式响应 → 工具在流中即刻执行 → 结果逐个 yield → UI 增量渲染，每一层都是流式的。

这意味着用户看到第一个字符的延迟 = API 首字节延迟，而不是 API 完整响应延迟 + 工具执行延迟 + 渲染延迟。在长回复场景下，差异可能是秒级的。

Security 安全模型的"三权分立"

Claude Code 的安全架构实现了真正的三权分立：规则系统（宪法 — 不可逾越的硬约束）+ 分类器（法官 — 基于上下文的灵活判断）+ 用户提示（陪审团 — 最终人类决策）。

特别巧妙的是 classifierApprovable 字段：某些安全检查可以被分类器在看到完整上下文后批准（如编辑 .claude/settings.json），而另一些永远不能（如 Windows 路径绕过攻击）。这在自动化便利性和安全底线之间找到了精确的平衡点。

Performance 命令式逃生舱是 React 的必修课

Claude Code 在 ScrollBox 中展示了一个重要的工程判断：不是所有 UI 更新都应该经过 React。高频滚动（每秒 60+ 事件）用命令式 scrollRef.current.scrollBy(delta) 直接修改，只在粘性状态变化这种低频异常时才走 React 状态更新。

配合 queueMicrotask 批量合并（10 个事件 → 1 次渲染），和 DECSTBM 硬件滚动优化（只发送滚动 delta 而非重绘整个视口），在终端环境中实现了流畅的 60 FPS 体验。

Design ToolSearch：当工具太多时的优雅降级

60+ 工具的完整 JSON Schema 会消耗大量初始 token。Claude Code 的解决方案很优雅：当工具数 > 40 时，只加载 4 个核心工具（Bash / FileRead / FileEdit / Agent），其余通过 ToolSearch 延迟发现。

AI 需要某个工具时，调用 ToolSearch("keyword")，系统返回匹配工具的完整 schema。代价是额外一轮 API 调用，收益是 30-50% 的初始 token 节省。这是典型的 "pay for what you use" 设计。

Innovation 编译时特性门控 = 零成本可选功能

Bun bundler 的 feature('KAIROS') 在编译时求值。如果特性未启用，整个 if 块（包括 require）被完全移除 — 不是运行时跳过，是物理上从产出包中删除。

这让 Claude Code 可以在同一代码库中维护内部版本（ANT_ONLY 特性）和外部版本，零运行时开销。已发现的特性门控包括：KAIROS（助手模式）/ COORDINATOR_MODE（多代理）/ REACTIVE_COMPACT / HISTORY_SNIP / TOKEN_BUDGET / BG_SESSIONS 等。

Engineering 遥测安全错误：强制 PII 审查的类型技巧

Claude Code 定义了一个故意名字很长的错误类：TelemetrySafeError_I_VERIFIED_THIS_IS_NOT_CODE_OR_FILEPATHS。这不是代码风格问题 — 这是一种类型系统强制的 PII 审查机制。

每次创建这个错误时，开发者必须看到这个名字，被提醒"我验证了这不是代码或文件路径"。类似的还有 AnalyticsMetadata_I_VERIFIED_THIS_IS_PII_TAGGED。把合规检查编码进类型名，让"做对的事"成为编译器强制的行为。
